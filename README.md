# 🌐 Static Website Deployment using GitHub Pages

This repository contains a simple static website that is successfully deployed using **GitHub Pages**.  
The main objective of this task is to understand how version control and free static hosting work together to publish a website on the internet.

This project demonstrates the complete workflow starting from creating a website, uploading it to GitHub, enabling GitHub Pages, and making the site live for public access.

## 📌 About the Project

A static website is a website that contains fixed content created using HTML, CSS, and JavaScript.  
Unlike dynamic websites, static websites do not require a database or backend server.

In this task, the website was built and deployed completely for free using GitHub Pages.

This project helped in understanding:
- How websites are hosted online
- How GitHub can be used beyond code storage
- How developers deploy small projects quickly

---

## 🛠️ Technologies Used

The following technologies were used to build and deploy this project:

- **HTML** → Structure of the website  
- **CSS** → Styling and layout  
- **JavaScript** → Basic interactivity  
- **Git** → Version control system  
- **GitHub** → Repository hosting  
- **GitHub Pages** → Free website hosting

---

## ⚙️ Deployment Steps Followed

1. Created a GitHub repository named **username.github.io**
2. Developed a simple static website using HTML, CSS and JavaScript
3. Uploaded the project files to the GitHub repository
4. Enabled GitHub Pages from repository settings
5. Selected branch: **main** and folder: **root**
6. Waited for GitHub to build and publish the site
7. Verified the live website using the GitHub Pages URL
